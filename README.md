# E-Commerce Order Management Database System

## Database Management System (DBMS) Semester Project

### Week 1 – Understanding the Business Problem

---

## Project Overview

The **E-Commerce Order Management Database System** is designed to manage the core operations of an online shopping platform. The system provides a centralized database for managing customers, product categories, products, suppliers, orders, payments, shipments, and customer reviews.

The primary objective of this project is to analyze the business requirements and prepare a Software Requirement Specification (SRS) document that serves as the foundation for the database design in subsequent project phases.

---

## Project Objectives

- Analyze the business requirements of an e-commerce system.
- Understand the business processes involved in online shopping.
- Identify stakeholders and their responsibilities.
- Define functional and non-functional requirements.
- Prepare a professional Software Requirement Specification (SRS).
- Create documentation for future database design.

---

## Business Scenario

A rapidly growing e-commerce company requires a centralized database system to efficiently manage its daily operations. The proposed system supports customer registration, product management, inventory tracking, order processing, payment management, shipment tracking, and customer reviews while reducing data redundancy and improving operational efficiency.

## Documents Description

### Requirement Analysis Report
Analyzes the business problem, project objectives, business processes, and explains the purpose of each core entity.

### Business Requirement Document (BRD)
Defines business objectives, stakeholders, business requirements, project scope, assumptions, constraints, and expected business benefits.

### Software Requirement Specification (SRS)
Provides detailed functional requirements, non-functional requirements, database requirements, business rules, system interfaces, and overall software specifications.
